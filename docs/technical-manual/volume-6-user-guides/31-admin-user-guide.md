## Introduction

### Purpose of This Guide

This comprehensive guide is designed for school administrators who manage the Southville 8B NHS Edge digital portal. Whether you're a principal, vice-principal, registrar, or administrative staff member, this guide will walk you through all the features and functions available in the administrator portal.

### Who Should Use This Guide

- **School Principals** - Overall system oversight and policy management
- **Vice-Principals** - Academic and student affairs management
- **Registrars** - Student enrollment and records management
- **Administrative Staff** - Day-to-day content and user management
- **IT Coordinators** - Technical configuration and system maintenance

### What You'll Learn

By the end of this guide, you will be able to:

- Navigate the administrator dashboard efficiently
- Manage user accounts for students, teachers, and staff
- Configure academic structures (sections, subjects, schedules)
- Create and publish content (news, events, announcements)
- Generate reports and analyze school data
- Configure system settings and permissions
- Troubleshoot common issues
- Perform bulk operations for efficiency

### Administrator Responsibilities

As an administrator, you are responsible for:

1. **User Account Management** - Creating, updating, and deactivating user accounts
2. **Academic Structure** - Setting up sections, subjects, and class schedules
3. **Content Oversight** - Publishing and moderating school content
4. **Data Integrity** - Ensuring accurate student and academic records
5. **System Configuration** - Managing settings and permissions
6. **Reporting** - Generating reports for school leadership and stakeholders
7. **Compliance** - Ensuring adherence to school policies and data privacy regulations

### Getting Help

If you encounter issues not covered in this guide:

- **Technical Support** - Contact your IT department or system administrator
- **Training Resources** - Access additional video tutorials and documentation
- **User Community** - Join the administrator forum for peer support
- **Emergency Contact** - For critical system issues, contact [emergency contact information]

## Getting Started

### Accessing the Administrator Portal

#### Step 1: Navigate to the Login Page

1. Open your web browser (Chrome, Firefox, Edge, or Safari recommended)
2. Go to the school portal URL: `https://southville8b.edu.ph`
3. You will see the homepage with a login button in the top-right corner

![Login page homepage](../images/admin-login-homepage.png)

#### Step 2: Enter Your Credentials

1. Click the **"Login"** button
2. Enter your administrator email address
3. Enter your password
4. Click **"Sign In"**

![Admin login form](../images/admin-login-form.png)

**Important Security Notes:**

- Never share your administrator credentials with anyone
- Use a strong, unique password (minimum 12 characters)
- Enable two-factor authentication if available
- Log out when using shared computers
- Report any suspicious account activity immediately

#### Step 3: First-Time Login Setup

If this is your first time logging in:

1. You will be prompted to change your default password
2. Enter a new strong password (follow the password requirements displayed)
3. Confirm your new password
4. Set up security questions for account recovery
5. Review and accept the administrator terms of service
6. Complete your profile information (phone number, department, photo)

![First-time setup wizard](../images/admin-first-login-setup.png)

### Understanding Your Admin Role

Different administrator accounts may have different permission levels:

- **Super Administrator** - Full system access, all permissions
- **Principal/Vice-Principal** - Academic and administrative oversight
- **Registrar** - Student records and enrollment focus
- **Content Manager** - News, events, and announcements management
- **Department Coordinator** - Department-specific management

Your role determines which menu options and features you can access. This guide covers all administrator functions; some sections may not apply to your specific role.

### Navigating the Interface

#### Main Navigation Menu

The administrator portal features a sidebar navigation menu with the following sections:

![Admin sidebar navigation](../images/admin-sidebar-menu.png)

1. **Dashboard** - Overview and quick statistics
2. **Users** - Student, teacher, and staff management
3. **Academics** - Sections, subjects, schedules, calendar
4. **Content** - News, events, announcements, clubs
5. **Reports** - Analytics and data reports
6. **Settings** - System configuration
7. **Profile** - Your account settings

#### Top Navigation Bar

The top bar includes:

- **Search Bar** - Quick search for students, teachers, or content
- **Notifications** - System alerts and important updates
- **Quick Actions** - Common tasks dropdown
- **Help** - Access to documentation and support

#### Using Keyboard Shortcuts

Speed up your workflow with these keyboard shortcuts:

- `Ctrl + K` - Open quick search
- `Ctrl + Shift + N` - Create new announcement
- `Ctrl + Shift + E` - Create new event
- `Ctrl + Shift + U` - Add new user
- `Esc` - Close modals and dialogs

### Customizing Your Dashboard

#### Personalizing Your View

1. Click on **Settings** in the sidebar
2. Navigate to **Dashboard Preferences**
3. Customize your dashboard widgets:
   - Drag and drop to reorder widgets
   - Toggle widgets on/off based on your needs
   - Resize widgets to focus on priority information
4. Click **Save Preferences**

![Dashboard customization](../images/admin-dashboard-customize.png)

#### Setting Notification Preferences

1. Go to **Profile** > **Notification Settings**
2. Choose which notifications you want to receive:
   - New student enrollments
   - Teacher submissions
   - System alerts
   - Content moderation requests
3. Select notification channels:
   - Email notifications
   - In-app notifications
   - SMS alerts (if configured)

5. Click **Update Preferences**

## User Management

### Overview of User Management

The user management section is one of the most critical areas of the administrator portal. Here you'll create, manage, and maintain user accounts for students, teachers, and other staff members.

![User management overview](../images/admin-users-overview.png)

### Managing Students

#### Viewing the Student Directory

1. Click **Users** in the sidebar
2. Select **Students** from the submenu
3. The student directory displays all enrolled students

The student list includes:
- Student ID number
- Full name
- Grade level and section
- Enrollment status (active, inactive, graduated)
- Contact information
- Last login date

#### Filtering and Searching Students

Use the filter panel to narrow down the student list:

**Filter by:**
- Grade level (Grade 7, 8, 9, 10)
- Section (specific class sections)
- Status (active, inactive, suspended, graduated)
- Enrollment year
- Gender
- Department or strand (for senior high school)

**Search options:**
- Student ID number
- Full name (first name, last name, or middle name)
- Parent/guardian name
- Email address
- Phone number

![Student filtering and search](../images/admin-student-filter.png)

#### Adding a New Student

**Step 1: Open the New Student Form**

1. Click **Users** > **Students**
2. Click the **"Add Student"** button (or use quick action)
3. The student enrollment form will open

**Step 2: Enter Basic Information**

Required fields:
- **First Name** - Student's first name
- **Middle Name** - Student's middle name (optional)
- **Last Name** - Student's last name/surname
- **Suffix** - Jr., Sr., III, etc. (optional)
- **Date of Birth** - MM/DD/YYYY format
- **Gender** - Male, Female, or Other
- **Student ID Number** - Auto-generated or manually entered

![Add student basic information](../images/admin-add-student-basic.png)

**Step 3: Academic Information**

- **Grade Level** - Select current grade (7, 8, 9, 10, 11, 12)
- **Section** - Assign to a class section
- **Track/Strand** - For senior high school (STEM, ABM, HUMSS, etc.)
- **Enrollment Status** - Active, Inactive, or Probation
- **School Year** - Current school year
- **LRN (Learner Reference Number)** - 12-digit unique ID

**Step 4: Contact Information**

Student contact:
- **Email Address** - Student's school email (auto-generated or custom)
- **Mobile Number** - Student's phone number (optional)
- **Address** - Complete home address
- **City/Municipality** - City of residence
- **Province** - Province
- **Postal Code** - ZIP code

Parent/Guardian contact:
- **Guardian Name** - Primary guardian's full name
- **Relationship** - Mother, Father, Legal Guardian, etc.
- **Guardian Email** - Contact email for parent
- **Guardian Phone** - Contact phone number
- **Emergency Contact** - Alternative emergency contact (optional)

![Add student contact information](../images/admin-add-student-contact.png)

**Step 5: Additional Information**

Optional fields:
- **Religion** - Religious affiliation
- **Nationality** - Student's citizenship
- **Previous School** - Last school attended
- **Special Needs** - Any learning accommodations required
- **Medical Conditions** - Important health information
- **Allergies** - Known allergies
- **Photo** - Upload student photo (JPEG or PNG, max 2MB)

**Step 6: Account Settings**

- **Username** - Auto-generated from name or custom
- **Temporary Password** - Auto-generated or set manually
- **Send Welcome Email** - Check to send login credentials via email
- **Account Activation** - Activate immediately or set activation date

**Step 7: Review and Submit**

1. Review all entered information for accuracy
2. Click **"Create Student Account"**
3. The system will validate the information
4. Upon success, you'll see a confirmation message
5. The student will receive a welcome email (if enabled)

![Student account creation success](../images/admin-student-created.png)

#### Editing Student Information

**To edit an existing student's information:**

1. Navigate to **Users** > **Students**
2. Search for or filter to find the student
3. Click on the student's name or click the **"Edit"** icon
4. The student profile editor will open
5. Make necessary changes to any field
6. Click **"Save Changes"**

**Important Notes:**
- Changes to student ID or LRN should be made carefully
- Email changes will require re-verification
- Grade level changes may affect section assignments
- All changes are logged in the audit trail

![Edit student profile](../images/admin-edit-student.png)

#### Bulk Student Import

For enrolling multiple students at once:

**Step 1: Download the CSV Template**

1. Go to **Users** > **Students**
2. Click **"Bulk Import"**
3. Click **"Download CSV Template"**
4. Save the template file to your computer

**Step 2: Prepare Your CSV File**

1. Open the template in Excel or Google Sheets
2. Fill in student information row by row
3. Follow the column header format exactly
4. Required columns: First Name, Last Name, Date of Birth, Grade Level
5. Save the file as CSV format

Example CSV format:
```
FirstName,MiddleName,LastName,DateOfBirth,Gender,GradeLevel,Section,Email,GuardianName,GuardianEmail,GuardianPhone
Juan,Santos,Dela Cruz,01/15/2010,Male,8,Einstein,jdelacruz@southville8b.edu.ph,Maria Dela Cruz,maria.dc@email.com,09171234567
Maria,Lopez,Garcia,03/22/2010,Female,8,Newton,mgarcia@southville8b.edu.ph,Pedro Garcia,pedro.g@email.com,09187654321
```

![CSV template example](../images/admin-csv-template.png)

**Step 3: Upload the CSV File**

1. Click **"Choose File"** or drag and drop your CSV file
2. The system will validate the file format
3. Review the preview of students to be imported
4. Check for any errors or warnings
5. Click **"Import Students"**

**Step 4: Review Import Results**

After import:
- **Successful Imports** - Number of students successfully added
- **Failed Imports** - Number of students that failed (with reasons)
- **Duplicate Entries** - Students already in the system
- **Download Error Report** - CSV file of failed imports for correction

![Bulk import results](../images/admin-bulk-import-results.png)

**Common Import Errors:**
- Missing required fields
- Invalid date format
- Duplicate student ID or email
- Invalid grade level or section
- Email format errors

#### Student Bulk Operations

Perform actions on multiple students at once:

**Step 1: Select Students**

1. Go to **Users** > **Students**
2. Use checkboxes to select students
3. Or use **"Select All"** to choose all filtered students

**Step 2: Choose Bulk Action**

Available bulk actions:
- **Activate Accounts** - Activate multiple inactive accounts
- **Deactivate Accounts** - Deactivate multiple accounts
- **Change Section** - Move students to a different section
- **Change Grade Level** - Promote or adjust grade levels
- **Export to CSV** - Export selected student data
- **Send Email** - Send a message to selected students
- **Reset Passwords** - Generate new temporary passwords
- **Delete Accounts** - Permanently remove accounts (use with caution)

**Step 3: Confirm and Execute**

1. Select your desired action from the dropdown
2. Configure action parameters (e.g., new section, email message)
3. Review the list of affected students
4. Click **"Execute Bulk Action"**
5. Confirm when prompted
6. Wait for the operation to complete

![Bulk operations interface](../images/admin-bulk-operations.png)

#### Viewing Student Details

Click on any student to view their complete profile:

**Profile Tabs:**

1. **Overview** - Basic information and quick stats
2. **Academic Records** - Grades, subjects, and performance
3. **Attendance** - Attendance history and patterns
4. **Activities** - Clubs, events, and extracurricular participation
5. **Assignments** - Assignment submissions and grades
6. **Behavior** - Disciplinary records and merits
7. **Family** - Parent/guardian contact information
8. **Documents** - Uploaded files and records
9. **Activity Log** - Account activity history

![Student profile details](../images/admin-student-profile.png)

#### Student Account Status Management

**Activating a Student Account:**

1. Open the student's profile
2. Click **"Account Status"** dropdown
3. Select **"Activate"**
4. Confirm the activation
5. The student can now log in

**Deactivating a Student Account:**

1. Open the student's profile
2. Click **"Account Status"** dropdown
3. Select **"Deactivate"**
4. Provide a reason (transfer, withdrawal, suspension, etc.)
5. Confirm the deactivation
6. The student will not be able to log in

**Suspending a Student Account:**

1. Open the student's profile
2. Click **"Account Status"** dropdown
3. Select **"Suspend"**
4. Set suspension period (start and end date)
5. Provide reason for suspension
6. Click **"Apply Suspension"**

**Graduating Students:**

1. Navigate to **Users** > **Students**
2. Filter by final grade level (e.g., Grade 10 or Grade 12)
3. Select students to graduate
4. Click **"Bulk Actions"** > **"Graduate Students"**
5. Select graduation year
6. Confirm graduation
7. Students will be marked as "Graduated" and accounts archived

### User Permissions and Roles

The system supports role-based access control:

1. **Super Administrator**
   - Full system access
   - User management
   - System configuration
   - All permissions

2. **Principal/Vice-Principal**
   - Academic oversight

### Emergency Procedures

#### System Outage

**Immediate Response:**

1. Verify the outage (check if it's your connection or system-wide)
2. Contact IT support immediately
3. Communicate with users:
   - Post notice on social media
   - Send email notification
   - Update phone message
4. Document:
   - Time of outage
   - Affected services
   - User impact
5. Follow IT team's instructions
6. Monitor restoration progress

**After Restoration:**

1. Verify all systems are functioning
2. Check data integrity
3. Test critical functions
4. Notify users that system is restored
5. Document incident and resolution
6. Review and improve outage procedures

#### Data Loss or Corruption

**Immediate Actions:**

1. Stop all user access to affected area
2. Contact IT support urgently
3. Do not attempt to fix data yourself
4. Preserve evidence (screenshots, error messages)
5. Identify last known good backup
6. Document:
   - What data is affected
   - When the problem was discovered
   - Potential cause

**Recovery:**

1. Work with IT to restore from backup
2. Verify restored data integrity
3. Test affected functions
4. Gradually restore user access
5. Monitor for recurring issues

**Prevention:**

1. Verify regular backups are working
2. Implement more frequent backups if needed
3. Train users on data safety
4. Review and strengthen data protection procedures

#### Security Breach

**If You Suspect a Breach:**

1. **Do Not Panic**
2. **Immediately:**
   - Document what you observed
   - Note the time
   - Take screenshots if safe to do so
3. **Contact:**
   - IT security team
   - School administration
   - Follow escalation procedures
4. **Do Not:**
   - Attempt to investigate on your own
   - Delete anything
   - Inform anyone except authorized personnel
   - Post about it on social media

**Follow IT Team Instructions:**

- May include changing passwords
- Restricting access temporarily
- Monitoring specific accounts
- Collecting evidence

**After Incident:**

1. Participate in post-incident review
2. Implement recommended security improvements
3. Update security procedures
4. Provide user security training if needed

## Troubleshooting

### Common Login Issues

#### Problem: Cannot Log In

**Possible Causes and Solutions:**

**1. Incorrect Password**

- Try typing password carefully (check Caps Lock)
- Use "Show Password" option to verify
- Click "Forgot Password" to reset

**2. Account Locked**

- Too many failed login attempts
- Wait 30 minutes or contact IT support
- Verify with IT that account is active

**3. Account Not Activated**

- New account may not be activated yet
- Check email for activation link
- Contact administrator to activate account

**4. Browser Issues**

- Clear browser cache and cookies
- Try a different browser
- Ensure browser is up to date
- Disable browser extensions temporarily

**5. Network Issues**

- Check internet connection
- Try accessing another website
- Contact IT if school network is down

#### Problem: Password Reset Not Working

**Steps to Resolve:**

1. Verify email address is correct
2. Check spam/junk folder for reset email
3. Ensure email account is accessible
4. Try password reset again after 5 minutes
5. If still not working, contact IT support directly

**Admin Action:**

1. Go to **Users**
2. Find the user
3. Click **"Reset Password"**
4. Generate new temporary password
5. Share password securely with user

### User Account Issues

#### Problem: User Cannot Find Their Account

**Troubleshooting:**

1. Search for user in admin panel
2. Check if account exists
3. Verify spelling of name and email
4. Check account status (active/inactive)

**If Account Doesn't Exist:**

1. Create new account
2. Send welcome credentials

**If Account is Inactive:**

1. Click on account
2. Change status to "Active"
3. Notify user

#### Problem: User Has Wrong Permissions

**Resolution:**

1. Go to **Users**
2. Find the user
3. Go to **"Permissions"** tab
4. Review current permissions
5. Update role or individual permissions
6. Click **"Save Changes"**
7. Ask user to log out and log back in

#### Problem: Duplicate User Accounts

**Resolution:**

1. Identify the correct account (most recent or most used)
2. Transfer any important data from duplicate to correct account
3. Deactivate or delete duplicate account
4. Update records to use correct account

### Content Issues

#### Problem: Content Not Appearing

**Check:**

1. **Publication Status**
   - Is content published or still in draft?
   - Is publication date in the future?

2. **Visibility Settings**
   - Who can see this content?
   - Does the viewer have permission?

3. **Content Location**
   - Is it in the right category?
   - Is it featured/unfeatured?

**Resolution:**

1. Edit the content
2. Verify status is "Published"
3. Check publication date is today or past
4. Verify visibility settings include intended audience
5. Save changes
6. Refresh page to verify

#### Problem: Images Not Displaying

**Causes and Solutions:**

**1. File Upload Failed**

- Re-upload the image
- Ensure file size is under limit
- Check file format (JPG, PNG accepted)

**2. Image Path Broken**

- Edit content
- Remove broken image
- Re-upload image
- Save content

**3. Browser Cache**

- Clear browser cache
- Hard refresh (Ctrl + Shift + R)
- Try different browser

#### Problem: Cannot Delete Content

**Reasons:**

1. **Content is Referenced Elsewhere**
   - Content is linked from other pages
   - Need to remove references first

2. **Insufficient Permissions**
   - User doesn't have delete permission
   - Need higher permission level

**Resolution:**

1. Verify you have delete permissions
2. Check for content references
3. Remove references
4. Try delete again
5. If still failing, contact IT support

### Academic Management Issues

#### Problem: Cannot Create Section

**Check:**

1. **Required Fields**
   - All required fields filled?
   - Valid grade level selected?

2. **Duplicate Section**

**4. Permission Issue**

4. Provide error message screenshots
